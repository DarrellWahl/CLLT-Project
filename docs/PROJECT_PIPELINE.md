## Pipeline Objective

Convert source road-sign manual pages into structured sign crops and a website-consumable dataset, with traceable metadata and verification artifacts.

## End-To-End Stages

## 1. Source Preparation

Inputs:

- PDF source and/or pre-rendered page PNGs.

Primary files:

- `assets/pdf/Road Traffic Signs.pdf`
- [assets/png/pages/page_1.png](../assets/png/pages/page_1.png) ... [assets/png/pages/page_5.png](../assets/png/pages/page_5.png)

Outputs:

- Stable page images for extraction and legend parsing.

## 2. Page Layout Detection

Primary implementation:

- `detect_page_layout()` in [scripts/extract_pipeline.py](../scripts/extract_pipeline.py)

Function:

- Determines content region and page type (`page_standard` vs `page5` legend behavior).

## 3. Candidate Region Detection

Primary implementation:

- `build_sign_mask()`, `detect_contour_boxes()`, `detect_edge_boxes()`

Function:

- Uses thresholding + HSV masks + morphology + edge detection.
- Produces candidate bounding boxes.

## 4. Merge And Filtering

Primary implementation:

- `merge_boxes()`, `reject_false_positives()`

Function:

- Merges overlapping/nearby candidates.
- Removes obvious non-sign regions using geometric and positional heuristics.

## 5. Candidate Structuring

Primary implementation:

- `build_candidates()`
- Dataclass: `SignCandidate`

Function:

- Converts filtered boxes into structured candidate entities with page context.

## 6. Optional OCR Text Integration

Primary implementation:

- `detect_text_blocks()`, `merge_text_boxes()`, `nearest_text_to_box()`, `extract_code_from_text()`

Function:

- Locates probable text/code zones and attempts code extraction.

Notes:

- OCR availability depends on runtime environment and tesseract stack.

## 7. Validation And Refinement

Primary implementation:

- `validate_candidate()`, `refine_candidate()`

Function:

- Detects edge-touching, too-small crops, and code-outside-crop issues.
- Applies controlled expansion retries.

## 8. Scoring And De-duplication

Primary implementation:

- `compute_confidence()`, `remove_duplicate_candidates()`

Function:

- Assigns confidence penalties based on warnings and missing OCR context.
- Removes duplicate candidates using IOU threshold.

## 9. Export And Documentation Artifacts

Primary implementation:

- `crop_image()`, `write_metadata()`, `write_stats()`, `draw_verification_image()`, `build_contact_sheet()`

Outputs:

- Sign crops (PNG)
- Metadata JSON/CSV (`metadata/page_*_signs.*`)
- Stats JSON ([logs/page_*_stats.json](../logs/))
- Verification overlays and contact sheets

## 10. Mapping And Website Dataset Publication

Supporting implementations:

- [scripts/parse_legend.py](../scripts/parse_legend.py)
- [scripts/map_signs.py](../scripts/map_signs.py)
- Additional mapping reconciliation processes

Publish target:

- [website/data/signs.json](../website/data/signs.json)

Website runtime consumes:

- [website/data/signs.json](../website/data/signs.json)
- `website/signs/*.png`

## Pipeline Controls

Central config:

- [assets/config/extraction_config.json](../assets/config/extraction_config.json)

Control categories:

- Merge/duplicate IOU behavior

## Known Gaps In Pipeline Closure

- Label semantic QA is not enforced as a mandatory gate.
- Multiple mapping variants can coexist without an authoritative resolver.
- No final publish manifest with integrity checks.

## Recommended Formal QA Gates

1. Schema validation gate for all JSON outputs.
2. Count reconciliation gate:
   - metadata total == mapping total == website dataset total
3. Semantic confidence gate with manual-review bucket.
4. Publish manifest gate with timestamp, source refs, and hash checks.
