## Priority Roadmap

## P0: Label Trustworthiness And Dataset Integrity

Goal:

- Promote current partially-labeled dataset into a trusted, auditable dataset.

Actions:

1. Define acceptance rules for code/description validity.
2. Build strict validator for:
   - code format validity
   - code-description pairing against legend
   - confidence thresholding
   - duplicate/contradiction detection
3. Split dataset into:
   - trusted_labeled
   - unlabeled
   - needs_review
4. Generate validation report with row-level reasons.

Acceptance criteria:

- 100% of trusted_labeled rows pass deterministic validation checks.
- No unresolved code-description contradictions in trusted_labeled.

## P1: Canonical Artifact Governance

Goal:

- Remove ambiguity from overlapping output trees.

Actions:

1. Define one canonical directory for each artifact class.
2. Add publish step that copies/promotes canonical artifacts only.
3. Mark deprecated directories/variants and archive obsolete files under `docs/archive/` references.

Acceptance criteria:

- Single documented source of truth for crops, mappings, and website payload.
- Rebuild process reproduces same artifact graph deterministically.

## P2: Automated QA And Regression Gates

Goal:

- Prevent regressions in extraction counts, schema, and runtime behavior.

Actions:

1. Add schema validators for JSON outputs.
2. Add reconciliation checks:
   - metadata totals
   - mapping totals
   - website dataset totals
3. Add smoke test for website render on full dataset.
4. Record baseline metrics and threshold alerts.

Acceptance criteria:

- Pipeline run fails fast on mismatch or invalid schema.
- Website smoke test passes in a repeatable environment.

## P3: Frontend Data Quality UX Enhancements

Goal:

- Make confidence and quality state explicit to end users.

Actions:

1. Expose review state badges in cards.
2. Add filter for trusted vs review-needed items.
3. Add optional labeled-only mode for quiz and library.

Acceptance criteria:

- Users can isolate trusted records without ambiguity.

## Suggested Execution Sequence

1. Complete P0 first.
2. Lock P1 governance immediately after P0 rules are final.
3. Implement P2 checks to enforce P0/P1 automatically.
4. Ship P3 UX refinements once data trust is formalized.

## Resourcing Notes

- Highest leverage is in validator/reconciliation tooling.
- Avoid additional mapping heuristics before validation framework exists.
- Keep OCR optional; never auto-promote OCR output without passing validation.
